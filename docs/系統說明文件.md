# 白頭翁整合照護 - 排班薪資管理系統

## 1. 系統概述

本系統為「愛立康有限公司」旗下「白頭翁整合照護」照服員排班與薪資管理系統，提供：

- 照服員資料管理
- 月曆式排班作業
- 薪資自動計算與手動調整
- 薪資明細表 Excel 匯出（符合公司既有格式）

**服務據點**：雙和、北醫

## 4. 資料庫模型

### 4.1 Employee（員工）

| 欄位 | 類型 | 說明 |
|------|------|------|
| id | Integer | 主鍵，自動遞增 |
| name | String(50) | 姓名 |
| emp_code | String(20) | 員編（如 2053） |
| start_date | Date | 到職日 |
| location | String(20) | 服務地點：雙和 / 北醫 |
| job_title | String(30) | 職稱，預設「照服員」 |
| bank_account | String(60) | 銀行帳號 |
| day_rate | Integer | 日班預設單價（元/天） |
| night_rate | Integer | 夜班預設單價（元/天） |
| is_active | Boolean | 是否在職，刪除時設為 False |

### 4.2 Schedule（排班）

| 欄位 | 類型 | 說明 |
|------|------|------|
| id | Integer | 主鍵 |
| employee_id | Integer | 外鍵 → Employee.id |
| date | Date | 日期 |
| shift | String(10) | 班別：**D**（日班）/ **N**（夜班）/ **OFF**（休假） |

唯一約束：(employee_id, date)，每人每天只能有一筆排班。

### 4.3 SalaryRecord（薪資記錄）

| 欄位 | 類型 | 說明 |
|------|------|------|
| id | Integer | 主鍵 |
| employee_id | Integer | 外鍵 → Employee.id |
| year | Integer | 年份（西元） |
| month | Integer | 月份 |
| day_shifts | Integer | 日班天數 |
| night_shifts | Integer | 夜班天數 |
| day_rate | Integer | 日班單價（可與員工預設不同） |
| night_rate | Integer | 夜班單價 |
| extra_earnings | Text (JSON) | 額外發金額，格式：`[{"name":"體檢費","amount":1510}]` |
| advance_pay | Integer | 扣：提早撥款本月薪資 |
| health_insurance | Integer | 扣：健保費眷屬 |
| welfare_fund | Integer | 扣：福利金 |
| leave_deduct | Integer | 扣：請假 |
| transfer_fee | Integer | 扣：匯費（通常 30 元） |
| extra_deductions | Text (JSON) | 額外扣金額，格式同 extra_earnings |
| note | Text | 備註 |
| payment_date | Date | 入帳日期 |

唯一約束：(employee_id, year, month)，每人每月一筆。

## 5. 功能模組說明

### 5.1 員工管理（/employees）

**功能**：新增、編輯、刪除照服員資料。

**操作方式**：
1. 點擊「新增員工」按鈕，填寫姓名、員編、地點、到職日、日/夜班單價、銀行帳號
2. 表格中點「編輯」可修改任何欄位
3. 「刪除」為軟刪除（標記為非在職），不會真正移除資料

**注意事項**：
- 不同據點有不同薪資標準，新增時需確認正確的日班/夜班單價
- 新人期間單價可能較低（如 2968），可透過薪資記錄手動調整

### 5.2 排班表（/schedule）

**功能**：以月曆表格方式管理全體照服員排班。

**操作方式**：
1. 上方可依「地點」篩選（雙和/北醫）
2. 左右箭頭切換月份（預設顯示上個月）
3. **點擊班別格子**即可切換：D → N → OFF → D（循環）
4. 底部自動統計每日 D 班與 N 班出勤人數
5. 右側顯示每人當月 D / N / OFF 合計天數

**班別定義**：

| 代碼 | 班別 | 時段 |
|------|------|------|
| D | 日班 | 08:00 - 20:00 |
| N | 夜班 | 20:00 - 08:00 |
| OFF | 休假 | - |

**畫面顯示**：
- 藍底 = D 班
- 黃底 = N 班
- 灰底 = OFF
- 淡紅底 = 週末（六、日）

### 5.3 薪資計算（/salary）

**功能**：依據排班自動計算薪資，支援手動調整加扣項目，匯出 Excel 薪資明細。

**計算邏輯**：

```
基本薪資 = (日班天數 × 日班單價) + (夜班天數 × 夜班單價)
發金額合計 = 基本薪資 + 額外加項（高配比、加班費、體檢費...）
扣金額合計 = 提早撥款 + 健保費眷屬 + 福利金 + 請假 + 匯費 + 其他扣款
實領薪資 = 發金額合計 - 扣金額合計
```

**優先順序**：
- 若該員工該月有**已儲存的薪資記錄**（SalaryRecord），系統使用記錄中的天數與單價
- 若無薪資記錄，系統從排班表自動計算天數，並使用員工預設單價

**操作方式**：
1. 左右箭頭切換月份
2. 每位員工顯示一張卡片，含日班、夜班、加項、扣項、實領金額
3. 點「編輯加扣項」可手動新增/修改：
   - **額外發金額**：如體檢費、高配比、加班費、新人差額
   - **固定扣金額**：提早撥款、健保費、福利金、請假、匯費
   - **額外扣金額**：如勞健保、超額扣款
   - 入帳日期、備註
4. 點「匯出明細」下載單人 Excel
5. 點「匯出全部薪資明細」下載全體 Excel（每人一個 Sheet）

**常見特殊處理**：

| 情境 | 處理方式 |
|------|---------|
| 新人期間較低單價 | 在薪資記錄中調低 day_rate，差額部分用「額外發金額」補正 |
| 一人多據點（如北醫+雙和） | 主要據點作為排班，其他據點天數以「額外發金額」項目加入 |
| 高配比加給 | 新增額外發金額項目，如「高配比 600」 |
| 6 小時加班 | 新增額外發金額項目，如「6hr 加班 1350」 |
| 勞健保扣款 | 填入「健保費眷屬」欄位，或新增額外扣金額 |
| 轉帳手續費 | 填入「匯費」欄位，通常 30 元 |

## 7. API 端點文件

### 7.1 員工 API

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/employees` | 取得全部在職員工，可加 `?location=雙和` 篩選 |
| POST | `/api/employees` | 新增員工，Body 為 JSON |
| PUT | `/api/employees/<id>` | 更新員工資料 |
| DELETE | `/api/employees/<id>` | 軟刪除員工 |

### 7.2 排班 API

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/schedule?year=2026&month=2` | 取得指定月份排班，可加 `&location=` |
| POST | `/api/schedule` | 設定單日班別 `{employee_id, year, month, day, shift}` |
| POST | `/api/schedule/batch` | 批次設定 `{employee_id, year, month, shifts: {"1":"D","2":"N"...}}` |

### 7.3 薪資 API

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/salary/calculate?year=2026&month=2` | 計算全體薪資 |
| POST | `/api/salary/save` | 儲存薪資記錄（加扣項調整） |
| GET | `/api/salary/export/<emp_id>?year=2026&month=2` | 匯出單人薪資明細 Excel |
| GET | `/api/salary/export-all?year=2026&month=2` | 匯出全體薪資明細 Excel |

## 8. 操作流程

### 8.1 每月排班與薪資作業流程

```
1. 員工異動          → /employees 新增或編輯員工資料
       ↓
2. 排班作業          → /schedule 點擊格子設定每人每日班別
       ↓
3. 薪資確認          → /salary 檢視自動計算結果
       ↓
4. 加扣項調整        → 點「編輯加扣項」設定高配比、加班費、勞健保等
       ↓
5. 匯出薪資明細      → 點「匯出明細」或「匯出全部」下載 Excel
       ↓
6. 發薪              → 將 Excel 作為薪資憑證
```

### 8.2 匯入歷史資料

若需從既有 Excel 排班表匯入資料，可參考 `import_salary.py` 腳本格式。

## 10. 注意事項

1. **資料庫位置**：`instance/salary.db`，請定期備份此檔案
2. **匯出目錄**：`exports/`，匯出的 Excel 會暫存於此
3. **安全性**：薪資為敏感資料，ngrok 對外連結用完請關閉（`pkill ngrok`）
4. **民國年轉換**：系統內部使用西元年，匯出 Excel 時自動轉為民國年（西元 - 1911）
5. **瀏覽器相容**：建議使用 Chrome 或 Edge 最新版
